--- layout: post title: How to compress multiple commits into one in Git by rebasing? date: '2012-04-23T20:20:00.000+02:00' author: Ondrej Platek tags: - Git - Linux - Techtips modified_time: '2012-07-23T17:16:05.266+02:00' blogger_id: tag:blogger.com,1999:blog-1883852367990943689.post-7004531357198457654 blogger_orig_url: http://oplatek.blogspot.com/2012/04/how-to-compress-multiple-commits-into.html --- Problem is that if you are not the maintainer of a repository merging creates
more complicated history than rebasing.


# Git merging history
    a - b - c
  /          \
1              Z
 \            /
    2 - 3 - 4


1 - a -2 -b -c -3 -4 -Z

Rebasing creates single line history which is easier to maintain
and further more you can compress your work into single commit by
following commands described at this blog

git log --oneline
git rebase -i SHA1


Nice trick is to use rebase --autosquash. Check help for more info!

Another solution is to use a merge command with --squash option.
